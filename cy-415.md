# cy-415 二手闲置物品交换平台

请根据以下要求生成项目代码和 README。

## 项目主要需求

### 1. 项目概述

**二手闲置物品交换平台（ReSwap）** 是一款基于以物换物模式的纯前端 Web 应用。用户可发布自己的闲置物品，浏览他人发布的物品，发起交换请求并管理交换记录。所有数据持久化存储在浏览器 localStorage / IndexedDB 中。项目类型：**Web 前端应用**。

### 2. 核心数据模型（≥ 3 个，必须独立成文件）

- **User（用户）**
  - 字段：id, nickname, avatar, phone, location, credit_score, created_at
  - 关键关系：一个用户可发布多个 Item，发起多个 Exchange
  - 业务动作：注册/登录（本地模拟）、修改资料、查看信用分
  - **此模型必须独立定义在 `src/models/user.ts`，禁止在组件内内联定义**

- **Item（闲置物品）**
  - 字段：id, user_id, title, description, category, condition（全新/九成新/八成新/战损）, images, status（可交换/已交换/已下架）, location, created_at
  - 关键关系：隶属于 User，被多个 Exchange 引用
  - 业务动作：发布物品、编辑、下架、筛选浏览
  - **此模型必须独立定义在 `src/models/item.ts`**

- **Exchange（交换请求）**
  - 字段：id, from_user_id, to_user_id, from_item_id, to_item_id, status（待确认/已同意/已拒绝/已完成）, message, created_at, updated_at
  - 关键关系：关联两个 User 和两个 Item
  - 业务动作：发起交换、同意/拒绝、确认完成、评价
  - **此模型必须独立定义在 `src/models/exchange.ts`**

### 3. 核心页面（≥ 4 个，至少 2 个页面共用同一业务组件）

- **/home（首页）**
  - 消费模型：Item
  - 关键交互：物品瀑布流展示、分类筛选、搜索、点击进入详情
  - 共享组件：`<ItemCard>`、`<CategoryFilter>`

- **/item/:id（物品详情）**
  - 消费模型：Item、User、Exchange
  - 关键交互：查看物品信息、查看物主资料、发起交换请求
  - 共享组件：`<ItemImageGallery>`、`<UserBrief>`

- **/publish（发布物品）**
  - 消费模型：Item
  - 关键交互：填写物品信息、上传图片（本地 base64）、选择分类和成色
  - 共享组件：`<ImageUploader>`、`<CategoryFilter>`（与 /home 共用）

- **/exchanges（交换管理）**
  - 消费模型：Exchange、Item、User
  - 关键交互：我发起的/我收到的交换列表、同意/拒绝/完成
  - 共享组件：`<ExchangeCard>`、`<EmptyState>`

- **/profile（个人中心）**
  - 消费模型：User、Item
  - 关键交互：修改资料、查看我发布的物品
  - 共享组件：`<UserBrief>`（与 /item/:id 共用）、`<AvatarUploader>`

### 4. 横切关注点（≥ 2 个）

- **主题切换**
  - 触达文件层：
    - `stores/themeStore.ts`
    - `constants/themes.ts`
    - 多个组件读取 theme 状态
    - `utils/themeUtils.ts`

- **全局错误处理/提示**
  - 触达文件层：
    - `utils/message.ts` 全局提示工具
    - `components/common/GlobalErrorBoundary.tsx`
    - 所有 API/Store 操作调用 message

### 5. 共享枚举/常量（≥ 2 组）

- **ItemStatus（物品状态）**
  - 值：AVAILABLE = 'available', EXCHANGED = 'exchanged', OFFLINE = 'offline'
  - 定义位置：`src/constants/item.ts`
  - 出现位置：Item 模型、ItemCard、物品详情、发布页、Exchange 逻辑、formatters
  - **README 必须列出该枚举的所有出现位置**

- **ExchangeStatus（交换状态）**
  - 值：PENDING = 'pending', ACCEPTED = 'accepted', REJECTED = 'rejected', COMPLETED = 'completed'
  - 定义位置：`src/constants/exchange.ts`
  - 出现位置：Exchange 模型、ExchangeCard、交换管理页、状态筛选器、formatters
  - **README 必须列出该枚举的所有出现位置**

### 6. 共享前端组件清单（≥ 3 个）

- `<ItemCard>`：被 /home 使用
- `<CategoryFilter>`：被 /home 和 /publish 共用
- `<ItemImageGallery>`：被 /item/:id 使用
- `<UserBrief>`：被 /item/:id 和 /profile 共用
- `<ExchangeCard>`：被 /exchanges 使用
- `<ImageUploader>`：被 /publish 使用
- `<EmptyState>`：被多个列表页共用
- `<AvatarUploader>`：被 /profile 使用

### 7. 共享 hooks/utils 清单（≥ 2 个）

- `useAuth()`：本地用户登录态管理
- `useLocalStorage()`：封装 localStorage 读写
- `useExchangeStats()`：交换状态统计
- `utils/storage.ts`：localStorage / IndexedDB 封装
- `utils/formatters.ts`：状态文本、日期格式化、信用等级文本

### 8. 状态管理要求

必须使用 Pinia 独立 store：
```
src/stores/
├── authStore.ts
├── itemStore.ts
├── exchangeStore.ts
└── themeStore.ts
```

禁止全部用组件内部 state。每个 store 按实体分文件，组件中按需引入。

### 9. 技术要求

- **英文短名（项目标识）**：`reswap`
- **前端端口**：18415
- **前端框架**：Vue 3 + TypeScript
- **UI 库**：Vant + Tailwind CSS
- **构建工具**：Vite
- **状态管理**：Pinia
- **路由**：Vue Router 4
- **数据持久化**：localStorage + IndexedDB（使用 idb-keyval 或 Dexie.js）
- **其他依赖**：dayjs、lodash-es

### 10. 文件结构强制清单

```
src/
├── api/              # 本地数据 API 层：userApi.ts, itemApi.ts, exchangeApi.ts（封装 storage 操作）
├── stores/           # authStore.ts, itemStore.ts, exchangeStore.ts, themeStore.ts
├── models/           # user.ts, item.ts, exchange.ts（独立数据模型类型定义）
├── types/            # 共享类型补充
├── components/common/# ItemCard, CategoryFilter, ItemImageGallery, UserBrief, ExchangeCard, ImageUploader, EmptyState, AvatarUploader
├── hooks/            # useAuth.ts, useLocalStorage.ts, useExchangeStats.ts
├── pages/            # Home, ItemDetail, Publish, Exchanges, Profile
├── router/           # index.ts + guards.ts
├── utils/            # storage.ts, formatters.ts, validators.ts
├── constants/        # item.ts, exchange.ts, themes.ts, messages.ts
└── App.vue
```

**严禁合并职责到单一文件。即使是简单功能，也必须按照上述分层结构拆分，确保后续每一个小改动都至少触达 3-5 个文件。**

### 11. 特殊约束：屎山代码设计（低内聚、高耦合、牵一发动全身）

1. **常量/工具类必须多处耦合**
   - `utils/formatters.ts` 同时包含日期格式化、状态文本、信用等级文本、成色文本
   - `constants/messages.ts` 同时包含页面提示文案、表单校验文案、日志式文案
   - 修改一个文案可能同时影响多个页面和组件

2. **枚举必须被多处引用**
   - ItemStatus 和 ExchangeStatus 必须在 models、constants、components、stores、formatters、router guards 中同时存在
   - 新增一个枚举值时，必须修改至少 8 处文件

3. **存储层必须单独管理但全应用引用**
   - `utils/storage.ts` 统一封装 localStorage / IndexedDB
   - 所有 store 和 api 层都必须通过 storage.ts 读写
   - storage.ts 中同时处理数据序列化、版本兼容、过期清理
   - 修改存储键名或结构需要同步修改所有 api 层

4. **全局状态必须多处读取**
   - themeStore 被 App.vue、多个页面、多个共享组件、ECharts/Vant 主题配置同时读取
   - authStore 被 router guards、导航栏、ItemCard（显示"我的"标签）、ExchangeCard 同时读取

5. **牵一发动全身的验证标准**
   - 后续如果要求"给 Item 新增一个 'booked' 状态"，必须触达：Item 模型、ItemStatus 枚举、itemStore 状态机、ItemCard、物品详情、发布页、Exchange 逻辑、formatters、messages、筛选器等 ≥ 8 个文件

### 12. 部署要求

这是纯前端 Web 应用，数据存储在浏览器本地，无需后端和 Docker。

前端代码放在项目根目录 `src/` 下：
- 页面、组件、路由、状态管理、API 请求、类型定义、样式和资源等放在对应的子文件夹

部署要求：
- 提供 `package.json`，包含 `scripts.dev` 和 `scripts.build`
- 开发运行命令：`pnpm dev` 或 `npm run dev`，访问 `http://localhost:18415`
- 构建命令：`pnpm build` 或 `npm run build`
- 生产部署可说明：将 `dist/` 目录通过 Nginx / 任意静态服务器托管

不需要 Docker Compose，但 README 中需说明：
- 如何安装依赖
- 如何启动开发服务器
- 如何构建生产包
